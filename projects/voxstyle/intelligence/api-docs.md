## Endpoints

### `GET /health`

Health check endpoint.

**Response**:
```json
{
  "ok": true,
  "gemini": true,
  "anthropic": false,
  "pexels": true,
  "pixabay": false,
  "unsplash": false
}
```
Values indicate which API keys are configured (not validated).

---

### `POST /generate-video`

Triggers a video generation pipeline run.

**Request**:
```json
{
  "subject": "The History of Artificial Intelligence",
  "topic": "optional topic refinement",
  "style": "optional style preference"
}
```

**Response**:
```json
{
  "jobId": "uuid-string",
  "status": "queued",
  "message": "Video generation started"
}
```

**Job Status Updates** (via `/video-status`):
| Status | Description |
|--------|-------------|
| `queued` | Job created, waiting to start |
| `processing` | Pipeline running (script → assets → halftone → voiceover → render) |
| `completed` | Video rendered to `out/video.mp4` |
| `failed` | Pipeline error |

---

### `GET /video-status/:jobId`

Returns current job status.

**Response**:
```json
{
  "jobId": "uuid-string",
  "status": "processing",
  "createdAt": "2026-07-22T10:00:00Z"
}
```

**404** if job not found.

---

### `GET /video-result/:jobId`

Returns completed job result (or error).

**Success Response**:
```json
{
  "jobId": "uuid-string",
  "status": "completed",
  "result": {
    "scenes": [ /* scene data */ ],
    "timepoints": [ /* audio timing */ ],
    "sceneCount": 20,
    "audioFiles": ["scene_0.mp3", "..."]
  }
}
```

**Error Response**:
```json
{
  "jobId": "uuid-string",
  "status": "failed",
  "error": "Gemini API error: invalid API key"
}
```

**404** if job not found.

## Error Handling

- **Pipeline errors**: Caught in `runPipeline()`, job marked as `failed` with error message
- **Missing API keys**: Health endpoint reports missing keys, pipeline falls back to alternatives
- **Python script errors**: `stderr` captured and included in pipeline error
- **Network errors**: Caught per external call, fallback to next source in chain
